# Проверка настройки с IP: 95.24.148.25

## Ваш внешний IP: 95.24.148.25

## Шаги для проверки:

### 1. Проверьте Dynamic DNS

Зайдите в настройки Dynamic DNS и убедитесь, что для `lms-solver.ddns.net` указан IP:
```
95.24.148.25
```

Если IP другой или устарел:
- Обновите IP в настройках Dynamic DNS
- Подождите 5-10 минут для распространения DNS

### 2. Проверьте доступность через прямой IP

Откройте в браузере (с другого устройства, не в вашей сети):
```
http://95.24.148.25:3000/api/health
```

**Результаты:**
- ✅ Работает → Проброс портов настроен правильно, проблема только в DNS
- ❌ Не работает → Проблема с пробросом портов или CGNAT

### 3. Проверьте через доменное имя

После обновления DNS (подождите 10 минут):
```
http://lms-solver.ddns.net:3000/api/health
```

### 4. Проверьте онлайн

Используйте сервис проверки портов:
```
https://www.yougetsignal.com/tools/open-ports/
```

Введите:
- **Remote Address:** 95.24.148.25
- **Port Number:** 3000
- Нажмите "Check"

**Результаты:**
- ✅ **Port is Open** → Проброс портов работает!
- ❌ **Port is Closed** → Проблема с пробросом портов или CGNAT

### 5. Проверьте настройки роутера

Убедитесь, что в правиле проброса портов:
- ✅ Внешний порт: **3000**
- ✅ Внутренний порт: **3000**
- ✅ Внутренний IP: **192.168.1.63** (IP вашего ПК)
- ✅ Протокол: **TCP**
- ✅ Правило включено

### 6. Проверьте IP вашего ПК

Убедитесь, что IP ПК не изменился:
```cmd
ipconfig
```

Найдите IPv4 Address. Если он изменился, обновите правило в роутере.

## Если порт закрыт:

### Возможные причины:

1. **CGNAT (Carrier-Grade NAT)**
   - Провайдер использует общий IP для нескольких клиентов
   - Проброс портов не работает
   - **Решение:** Попросить у провайдера публичный IP (обычно платно)

2. **Проброс портов не настроен**
   - Проверьте настройки роутера
   - Убедитесь, что правило активно

3. **Брандмауэр блокирует**
   - Проверьте правило Windows Firewall (уже создано)
   - Проверьте брандмауэр роутера

## Быстрый тест:

Попробуйте открыть с телефона (через мобильный интернет, НЕ Wi-Fi):
```
http://95.24.148.25:3000/api/health
```

Если работает - значит проброс портов настроен правильно!

```

Или временно (пока DNS не обновился):
```
