# Устранение блокировки IP-адреса роутером/провайдером

## Проблема: Роутер/провайдер блокирует весь трафик к IP-адресу

Если ping не проходит и все порты недоступны, но на мобильной сети все работает - роутер или провайдер блокирует IP-адрес `130.61.200.70`.

## Решения:

### Решение 1: Проверить настройки роутера

1. **Откройте веб-интерфейс роутера:**
   - Обычно `192.168.1.1` или `192.168.0.1`
   - Логин/пароль обычно на наклейке роутера

2. **Проверьте настройки родительского контроля:**
   - Отключите временно для теста
   - Или добавьте `130.61.200.70` в белый список

3. **Проверьте настройки файрвола:**
   - Отключите временно для теста
   - Или добавьте правило для `130.61.200.70`

4. **Проверьте настройки фильтрации контента:**
   - Отключите временно для теста
   - Или добавьте исключение

### Решение 2: Использовать домен вместо IP

Иногда роутеры/провайдеры блокируют IP-адреса, но не домены. Можно настроить домен:

#### Вариант A: Бесплатный домен через DuckDNS или No-IP

1. Зарегистрируйтесь на [DuckDNS](https://www.duckdns.org/) или [No-IP](https://www.noip.com/)
2. Создайте домен (например, `lms-api.duckdns.org`)
3. Настройте динамический DNS на сервере
4. Обновите URL в расширении на домен

#### Вариант B: Использовать Cloudflare (рекомендуется)

1. Зарегистрируйте домен (или используйте бесплатный)
2. Добавьте домен в Cloudflare
3. Создайте A-запись, указывающую на `130.61.200.70`
4. Используйте домен вместо IP

### Решение 3: Использовать VPN

Если ничего не помогает, можно использовать VPN:

1. **Установите VPN на ПК** (например, ProtonVPN, Windscribe - есть бесплатные варианты)
2. Подключитесь к VPN
3. Расширение будет работать через VPN

### Решение 4: Обратиться к провайдеру

Позвоните в техподдержку провайдера Wi-Fi и уточните:
- Не блокируют ли они доступ к `130.61.200.70`
- Можно ли разблокировать этот IP-адрес
- Есть ли у них фильтрация трафика

### Решение 5: Использовать прокси

Можно настроить прокси для расширения Chrome:

1. Настройте прокси-сервер (например, через SSH туннель)
2. Настройте Chrome использовать прокси для расширения

### Решение 6: Изменить IP-адрес сервера (если возможно)

Если у вас есть возможность изменить IP-адрес в Oracle Cloud:
1. Создайте новый instance с новым IP
2. Или используйте Load Balancer с другим IP

## Рекомендуемое решение: Использовать домен

Домены реже блокируются, чем IP-адреса. Можно использовать бесплатный домен через DuckDNS.

### Быстрая настройка DuckDNS:

1. Зарегистрируйтесь на https://www.duckdns.org/
2. Создайте домен (например, `lms-api`)
3. Получите токен
4. На сервере установите обновление IP:

```bash
# Установите curl (если нет)
sudo apt-get install -y curl

# Создайте скрипт обновления
sudo nano /usr/local/bin/update-duckdns.sh
```

Вставьте:
```bash
#!/bin/bash
echo url="https://www.duckdns.org/update?domains=YOUR_DOMAIN&token=YOUR_TOKEN&ip=" | curl -k -o /tmp/duckdns.log -K -
```

```bash
# Сделайте исполняемым
sudo chmod +x /usr/local/bin/update-duckdns.sh

# Добавьте в cron для автоматического обновления
sudo crontab -e
```

Добавьте:
```
*/5 * * * * /usr/local/bin/update-duckdns.sh >/dev/null 2>&1
```

5. Обновите URL в расширении на `http://YOUR_DOMAIN.duckdns.org:3000`

## Временное решение: Использовать мобильный интернет

Пока проблема не решена, можно:
1. Подключить ПК к мобильной точке доступа
2. Или использовать USB-модем с мобильным интернетом

## Проверка настроек роутера

Попробуйте зайти в настройки роутера и проверить:

1. **Родительский контроль** - отключите временно
2. **Файрвол** - отключите временно
3. **Фильтрация контента** - отключите временно
4. **Блокировка IP** - проверьте черный список

Если после отключения заработало - проблема в настройках роутера.
