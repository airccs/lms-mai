# Быстрый старт сервера

## Где находится сервер?

Сервер находится в папке `server/` вашего проекта:
```
f:\lms\
  └── server\
      ├── server.js      ← Основной файл сервера
      ├── package.json   ← Зависимости
      └── README.md      ← Документация
```

## Как запустить сервер?

### Вариант 1: Локальный запуск (для тестирования)

1. **Установите Node.js** (если еще не установлен)
   - Скачайте с https://nodejs.org/
   - Версия 14 или выше

2. **Откройте терминал/командную строку** в папке `server`:
   ```bash
   cd f:\lms\server
   ```

3. **Установите зависимости**:
   ```bash
   npm install
   ```

4. **Запустите сервер**:
   ```bash
   npm start
   ```

   Сервер запустится на `http://localhost:3000`

5. **Настройте расширение**:
   - Откройте popup расширения
   - Включите "Синхронизацию с сервером"
   - Введите URL: `http://localhost:3000`
   - Сохраните настройки

### Вариант 2: Развертывание в интернете

Для работы с другими пользователями нужно развернуть сервер в интернете.

#### Бесплатные варианты:

1. **Heroku** (бесплатный тариф):
   ```bash
   # Установите Heroku CLI
   heroku create ваш-проект
   git push heroku main
   ```

2. **Railway** (бесплатный тариф):
   - Загрузите код на GitHub
   - Подключите к Railway
   - Автоматическое развертывание

3. **Render** (бесплатный тариф):
   - Создайте Web Service
   - Подключите репозиторий
   - Автоматическое развертывание

4. **Vercel** или **Netlify** (для serverless функций)

#### Платные варианты:

- **VPS** (DigitalOcean, AWS, Azure)
- **Dedicated Server**

## Проверка работы сервера

После запуска откройте в браузере:
```
http://localhost:3000/api/health
```

Должен вернуться ответ:
```json
{
  "status": "ok",
  "message": "Quiz Solver API is running"
}
```

## Важно!

⚠️ **Текущий сервер хранит данные в памяти** - при перезапуске все данные теряются!

Для продакшена нужно:
- Использовать базу данных (MongoDB, PostgreSQL)
- Добавить персистентное хранилище
- Настроить резервное копирование
